## Mini Shell

Mini Shell - это командная оболочка, написанная на Python, которая воспроизводит основные команды терминала Linux. Проект предоставляет платформу для выполнения файловых операций с цветным выводом, ures fliendly сообщениями при недопустимом вводе и системой логирования.

## Файловые операции

*   **`ls`** - список файлов и папок
    * `-l` - подробная информация (права доступа, размер, дата изменения)
*   **`cd`** - смена директории
*   **`cat`** - просмотр содержимого файлов
*   **`cp`** - копирование файлов и папок
    * `-r` - рекурсивное копирование директорий
*   **`mv`** - перемещение и переименование файлов
*   **`rm`** - удаление файлов и папок

## Поиск по файлам

*   **`grep`** - поиск текста в файлах
    * `-r` - рекурсивный поиск в директориях
    * `-i` - поиск без учета регистра

## Архивация/распаковка

*   **`zip`** - создание zip архивов
*   **`unzip`** - распаковка zip архивов
*   **`tar`** - создание tar архивов
*   **`untar`** - распаковка tar архивов

## Принятые решения

Для нектороых команд (`ls`, `cat`, `rm`, `zip`, `unzip`, `tar`, `untar`) возможен ввод нескольких аргументов, в таком случае операции будут выполнены последовательно. Например, за вводом `zip path1 path2 path3` последует zip архивация файлов, находящихся по путям `path1`, `path2` и `path3` поочередно.

Также для большинства функций, принимающих на вход путь директории, поддерживается ввод символа `.` для быстрого ввода текущей директории. Например, если текущая директория `C:\Users\admin`, то ввод `<comand> .` равносилен вводу `<comand> C:\Users\admin`.

## Обработка ошибок

Программа использует комплексную обработку ошибок:
*   Проверка существования файлов и директорий
*   Проверка прав доступа
*   Валидация аргументов команд
*   User friendly сообщения при вводе с ошибкой

## Техничесие особенности

*   Программа Написана на Python с использованием модулей os и shutil
*   Поддерживает мультиплатформенную работу (Windows, Linux, macOS)
*   Включает систему логирования операций
*   Покрыта тестами функций более чем на 80% (тестирование с помощью pytest)

## Параметры ввода

*   На вход программе пользователь подает строку, первое слово строки - название команды, которую необходимо выполнить
*   Далее строка должна осдержать путь к файлу (абсолютный или относительный).
*   Если программа предполагает работу с несколькими файлами или директориями (например, команда cp), то пути стоит вводить последовательно, разделяя пробелами (одним или несколькими).
*   Если путь содержит пробел, то весь путь или его часть, включающую этот пробел, стоит поместить в кавычки: '' или ""
Например, `cd 'C:\Users\admin\Новая папка'` или `cd C:\Users\admin\"Новая папка"`
*   Флаги `-r`, `-l` и `-i`, если они предполагаются функцией, могут записываться в любом порядке и месте, кроме начала строки. Отделяются пробелами (одним или несколькими)

## Обработка ввода

*   Множественные аргументы - возможность обработки нескольких путей
*   Пробелы в путях - пробелы, расположенные между кавычками не разделяют
*   Специальные символы:
    * `.` - текущая директория
    * `~` - домашняя директория
    * `..` - переход на уровень выше
